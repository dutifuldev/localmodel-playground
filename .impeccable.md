# Design Context

Target audience: developers, prompt engineers, and local-model evaluators working at a desk with LM Studio, Ollama, vLLM, or OpenAI-compatible endpoints.

Use cases: construct API requests, load and edit JSON prompt files, run local models, compare outputs, inspect raw request/response data, and keep prompt artifacts version controlled.

Brand personality/tone: quiet, utilitarian, precise, and familiar to OpenAI Playground users. The interface should feel like a production developer tool, not a marketing page.
